# 第28回：回帰と勾配降下法（機械学習）

みなみ先生：（明るく）はい、「みなみ先生の数学・機械学習ラジオ」、今日も始めていきましょう。

たくみ：お願いします。前回は機械学習全体の枠組みでしたよね。教師あり学習とか教師なし学習とか、訓練データと検証データとテストデータに分ける話。正解率や適合率、再現率もたくさん出てきて、正直まだ頭がごちゃっとしてます。

みなみ先生：（笑いながら）初回はどうしても地図を広げる回だからね。あのとき「正則化の中身は次の単元で」って言ったの、覚えてる？

たくみ：あ、それ気になってました。

みなみ先生：（にっこり）今日はまさにそこ。教師あり学習の第一弾、数値を予測する**回帰**と、その裏側でパラメータを求めている**勾配降下法**、それにリッジ回帰まで一気にやろう。

たくみ：（少し身構えて）勾配降下法……名前だけで難しそうです。

みなみ先生：（軽く笑って）大丈夫、数学Bの「数学と社会生活」でやった回帰直線の延長線上にある話だよ。今日のゴールは3つ。線形回帰と損失関数の意味を理解すること、勾配降下法の更新式を自分の手で導出できること、そしてリッジ回帰の効果を数式から読み取れること。

たくみ：数値を予測する式そのものより、その式の中身をどう決めるか、っていう話なんですね。

みなみ先生：そのとおり。始めよう。

たくみ：お願いします！

みなみ先生：入力エックスから数値ワイを予測する一番シンプルなモデルはこう。予測値は、傾きをダブリュー、切片をビーとして「ダブリューかけるエックス、足すビー」。中学でやった一次関数と同じ形だよ。

たくみ：それなら分かります。

みなみ先生：数学Bでは、残差の2乗和を最小にする直線を、微分してイコールゼロと置くことで一発で求めたよね。正規方程式を解く方法。でも特徴量が何十個にもなると、その計算がどんどん重くなる。そこで、損失を少しずつ小さくする方向にパラメータを更新していく、**勾配降下法**が登場する。

たくみ：一発で解くのをあきらめて、ちょっとずつ近づいていく作戦ですか。

みなみ先生：まさに。良し悪しを測る物差しには、平均二乗誤差、エムエスイーを使う。実際の値と予測値の差を2乗して、全データで平均を取ったもの。

たくみ：差を2乗して平均、ですね。

みなみ先生：（少し声を落として）ここが山場。この損失を、ダブリューで偏微分したものと、ビーで偏微分したものを求める。計算していくと、エルをダブリューで偏微分したものは、「予測値と実際の値の差にエックスをかけたものを全データで足し合わせて、エヌ分の2倍する」という形になる。ビーの方は、エックスをかける部分がないだけで同じ形。

たくみ：（うーんと唸って）つまり「予測が外れている分だけ、その方向にずれを足し合わせたもの」ってことですか。

みなみ先生：（嬉しそうに）いい感覚。この2つをセットにしたものが「勾配」で、これは「損失が最も急に増加する方向」を向いている。だから逆向きに少し動かせば、損失は小さくなる。動かす量を調整する係数がイータ、これを**学習率**と呼ぶ。更新式は、ダブリューを、今のダブリューからイータかける偏微分値を引いた値に置き換える。ビーも同様。これを損失がほとんど変わらなくなるまで繰り返す。

たくみ：一歩ずつ、谷底に向かって下りていくイメージですね。

みなみ先生：（笑って）まさに山下り。しかもエムエスイーはお椀型の凸関数だから、繰り返せば必ず唯一の最小値、さっきの正規方程式の解と同じ最適な直線にたどり着く。ただし学習率が大きすぎると谷底を通り過ぎて振動したり損失が増えていってしまい、小さすぎると1回の動きがわずかで、ものすごい反復回数が必要になる。

たくみ：（ため息まじりに）大股すぎても小股すぎてもダメなんですね。

みなみ先生：じゃあ手を動かそう。データは、エックスとワイの組で1と2、2と3、3と5。ダブリューはゼロ、ビーもゼロから、学習率0.1で1ステップやってみる。

たくみ：ゼロからスタートですね。

みなみ先生：ダブリューとビーがゼロだと予測値は全部ゼロ。だから予測値マイナス実際の値は、マイナス2、マイナス3、マイナス5になる。ここからエルをダブリューで偏微分したものは、3分の2かける「マイナス2かける1、足すマイナス3かける2、足すマイナス5かける3」。

たくみ：（電卓を叩くように）マイナス2、マイナス6、マイナス15を足してマイナス23、それを3分の2倍すると……マイナス15.33くらいです。

みなみ先生：正解。エルをビーで偏微分したものは、3分の2かける「マイナス2足すマイナス3足すマイナス5」で、だいたいマイナス6.67。じゃあ更新しよう。新しいダブリューは、ゼロから0.1かけるマイナス15.33を引いてだいたい1.533。新しいビーは、ゼロから0.1かけるマイナス6.67を引いてだいたい0.667。

たくみ：（驚いた声で）え、1回動かしただけでダブリューが1.5くらいまで進むんですか。

みなみ先生：（にっこり）このデータ、数学Bのやり方で求めると最小二乗解はダブリューがだいたい1.5、ビーがだいたい0.33。たった1ステップでもうほとんど正解に近いところまで来てる。損失でも確認すると、更新前のエムエスイーは2乗誤差4と9と25の平均でだいたい12.67。更新後は誤差が0.2、0.73、0.27くらいになって、エムエスイーはだいたい0.216。

たくみ：（感動した様子で）12.67から0.216、ものすごく減ってますね。谷底に向かってちゃんと下りてるのが実感できました。

みなみ先生：じゃあもうひとつの柱、リッジ回帰。過学習を防ぐために、損失に「係数の2乗に比例するペナルティ」を加える方法で、エル2正則化とも呼ばれる。話を簡単にするため、切片ビーを除いた1変数で考えるよ。損失は「実際の値と予測値の差の2乗の合計」に、ラムダかけるダブリューの2乗を足したもの。ラムダが正則化の強さを表すハイパーパラメーター。

たくみ：さっき出てきた正則化、これですね。

みなみ先生：これをダブリューで微分してイコールゼロと置いて整理すると、ダブリューのリッジ解は、エックスとワイの積和を、エックスの2乗和にラムダを足したもので割った形になる。ラムダがゼロなら普通の最小二乗解に一致して、ラムダを大きくするほど分母が大きくなって、ダブリューはどんどんゼロに近づく。これを**縮小推定**と呼ぶ。

たくみ：分母にラムダが足されるだけで、そんなに効くんですね。

みなみ先生：数字で確認しよう。積和が3、2乗和が2だとすると、ラムダがゼロならダブリューは2分の3で1.5。ラムダが2なら分母が4になって4分の3で0.75。ラムダが8なら分母が10になって10分の3で0.3。

たくみ：（納得した様子で）1.5、0.75、0.3……本当にゼロに近づいてる。

みなみ先生：（少し真剣な声で）でも、ラムダを大きくしすぎるとダブリューがほとんどゼロに潰れて、データの傾向を無視した**未学習**のモデルになる。正則化は過学習を防ぐためのものなのに、やりすぎると逆に予測が使い物にならなくなる、これは本当に多くの人がやりがちな失敗。

たくみ：（ヒヤッとした様子で）ちょうどいい強さを探さないといけないんですね。

みなみ先生：あと2つだけ注意点。ひとつ、正則化のペナルティには普通、切片ビーは含めない。ビーはデータ全体の水準を表すだけで、過学習の原因にはなりにくいから。もうひとつ、特徴量ごとにスケールが大きく違うと、勾配降下法の収束が悪くなったり、正則化の効き方が不公平になったりする。だから平均をゼロ、バラつきを1に揃える**標準化**を先にやっておくのが大事。

たくみ：（少し焦った声で）単位が違う量をそのまま同じ式に突っ込んじゃいけない、ってことですね。気をつけます。

みなみ先生：（優しく）それと最後にひとつ。今日のエムエスイーと線形モデルの組み合わせは凸関数だから必ず最適解に着いたけど、これから出てくるニューラルネットワークのような複雑な関数では、途中の谷、局所最適に迷い込む可能性もある。「必ず最適解に着く」は今日の話だからこそ言えることなんだ。

たくみ：分かりました。今日は数式を実際に手で追いかけられたのが、理解の助けになりました。

みなみ先生：（笑顔で）じゃあまとめ。線形回帰はダブリューエックス足すビーという式で予測し、平均二乗誤差で良し悪しを測る。勾配降下法は、損失の勾配の逆向きに、学習率の分だけ少しずつパラメータを動かして損失を下げる方法。そしてリッジ回帰は、係数の2乗にペナルティを課すことで係数をゼロに向かって縮小させ、過学習を防いでくれる。

たくみ：（達成感のある声で）今日もありがとうございました！

みなみ先生：次回は、今日の「数値を予測する回帰」に対して、「カテゴリーを予測する分類」に入っていくよ。ロジスティック回帰、決定木、ケー近傍法という3つの手法を見ていく予定。

たくみ：（目を輝かせて）分類、楽しみです！ 迷惑メール判定みたいな話ですよね。

みなみ先生：（笑って）まさにそれ。それじゃあ今日はここまで、お疲れさま！

たくみ：ありがとうございました！
