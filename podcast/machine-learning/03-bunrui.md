# 第29回：分類（機械学習）

みなみ先生：（明るく）はい、「みなみ先生の数学・機械学習ラジオ」、今日も始めていきましょう。前回は教師あり学習の一つ目、回帰の話だったね。直線を当てはめて、二乗誤差を損失関数にして、勾配降下法でパラメータを少しずつ更新していく、っていう流れをやったよね。

たくみ：（少し誇らしげに）はい、坂道を下るみたいに、損失が小さくなる方向にちょっとずつ進んでいく、っていうイメージ、すごく腑に落ちました。

みなみ先生：（うなずいて）良かった。回帰は「数値そのもの」を予測する話だったけど、今日はその隣にあるもう一つの教師あり学習、「分類」がテーマだよ。病気か健康か、迷惑メールかそうでないか、みたいに、カテゴリーを当てる問題ね。

たくみ：（興味深そうに）分類かあ。イエスかノーかを当てる、みたいな感じですか。

みなみ先生：そういうこと。今日はその代表選手として、ロジスティック回帰、決定木、k近傍法っていう3つの手法をやっていくよ。それぞれ考え方が全然違うから、比べながら聞くと面白いと思う。

たくみ：（気合を入れて）お願いします。

みなみ先生：まずロジスティック回帰から。名前に「回帰」って入ってるけど、これは分類の手法なの。

たくみ：（驚いた声で）え、回帰なのに分類なんですか。ややこしい。

みなみ先生：（笑いながら）そうなんだよね、名前だけ見ると混乱するポイント。2値分類、つまり陽性か陰性かを当てたいとき、前回みたいにいきなり実数を出力しても扱いにくいの。だから、出力を「確率」、つまりゼロから1の間の値にする工夫をする。まず前回と同じように、ダブリューかけるエックス足すビー、っていう線形な計算をしてゼットを作る。これを「シグモイド関数」っていう関数に通すんだ。

たくみ：シグモイド関数って、どんな形の関数ですか。

みなみ先生：ゼットが大きい正の値ならほぼ1に近づいて、ゼットが大きい負の値ならほぼゼロに近づく。そして、ゼットがちょうどゼロのときは、きっかり0.5になる。グラフに描くと、アルファベットのエスを横に伸ばしたような、なめらかなカーブになるよ。

たくみ：（あ、という声で）ゼロから1の間しか出ないから、確率として使えるってことですね。

みなみ先生：そのとおり。この確率、予測確率が0.5以上、つまりゼットがゼロ以上のときに「陽性」、そうでなければ「陰性」って判定するのが基本のルール。これを「決定境界」って呼ぶよ。

たくみ：なるほど、確率を求めてから、最後にイエスかノーかに変換するんですね。

みなみ先生：うん。それから、このシグモイド関数、実は微分がすごくきれいな形になるの。シグモイド関数を微分すると、シグモイド関数そのものかける、1引くシグモイド関数、っていう形になる。

たくみ：（少し混乱した声で）自分自身を使って自分の微分が書けちゃうんですか。

みなみ先生：そう、不思議だよね。この性質、実はこの先ニューラルネットワークの学習でも何度も出てくる大事な式だから、名前だけでも覚えておいて損はないよ。

たくみ：覚えておきます。

みなみ先生：次に、損失関数の話。回帰では二乗誤差を使ったけど、分類では「交差エントロピー損失」っていう別の損失を使うの。正解ラベルが1のデータには、予測確率の対数にマイナスをつけたものが、正解ラベルがゼロのデータには、1引く予測確率の対数にマイナスをつけたものが、それぞれ足し合わされる形。

たくみ：（少し不安そうに）対数が出てくると身構えちゃいます……。

みなみ先生：大丈夫、感覚で説明するね。予測確率が正解にすごく近ければ、この値はほとんどゼロになる。でも、正解から外れて、しかも自信満々に間違えているほど、この値は急激に大きくなる。つまり「自信満々の間違い」に強いペナルティを与える仕組みなんだ。

たくみ：（納得した声で）ただ間違えるだけじゃなくて、自信の度合いまで見てるってことですね。

みなみ先生：そういうこと。次は決定木。これは全然違うアプローチで、「ある特徴量がしきい値以上か未満か」で枝分かれを繰り返して、木の形でデータを分類していく手法。

たくみ：フローチャートみたいなイメージですか。

みなみ先生：まさにそう。ただ、どの特徴量、どのしきい値で分岐すれば賢い木になるかを決めなきゃいけない。そこで使うのが「ジニ不純度」っていう指標。あるノードの中で、クラスの混ざり具合を数値にしたものね。全員が同じクラスならジニ不純度はゼロ、つまり最も純粋。逆に、クラスが半々に混ざっているほど、ジニ不純度は大きくなる。

たくみ：混ざり具合を数字で測るんですね。

みなみ先生：分岐の前と後で、このジニ不純度がどれだけ下がったか、その差を「情報利得」って呼ぶの。情報利得が一番大きくなる分岐を選ぶことで、賢い木を育てていくんだ。

たくみ：分岐するたびに、一番「スッキリする」分け方を選んでいくってことですね。

みなみ先生：そのとおり。最後にk近傍法。これは学習という手続きをほとんどしない、すごくシンプルな手法。新しいデータが来たら、訓練データの中から距離が近いケー個を探して、その多数決でラベルを決めるだけ。

たくみ：（拍子抜けした声で）え、それだけなんですか。

みなみ先生：（笑いながら）それだけ。ただ、このケーの選び方が重要で、ケーが小さいほど境界が複雑になって過学習しやすくなる。逆にケーが大きいほど境界がなめらかになりすぎて、未学習に近づく。これも前回話したバイアスとバリアンスのトレードオフの一例だよ。

たくみ：前回の話、ここでもつながるんですね。

みなみ先生：うん。それと、距離を測る手法だから、特徴量のスケールを揃える、つまり標準化しておくことが特に大事。それと関連して、複数のモデルを組み合わせる「アンサンブル学習」も少しだけ触れておくね。訓練データからランダムに抽出したデータごとに別々の木を作って多数決を取る「バギング」は、各木のバリアンスを下げる効果がある。分岐のたびに使う特徴量もランダムに絞ったものが「ランダムフォレスト」。逆に、前のモデルの誤りを次のモデルが重点的に学習していく「ブースティング」は、バイアスを下げる効果があるんだ。

たくみ：バギングはバリアンス、ブースティングはバイアス、って対になってるんですね。

みなみ先生：そういうイメージで覚えておくといいよ。じゃあ、実際に数字を使って声で解いてみよう。まずシグモイド関数。ゼットが2のとき、シグモイド関数の値を求めてみて。イーのマイナス2乗はだいたい0.135とするよ。

たくみ：えっと、シグモイド関数は、1足すイーのマイナスゼット乗ぶんの1、でしたよね。ゼットが2だから、1足す0.135ぶんの1。

みなみ先生：計算すると？

たくみ：1.135ぶんの1……だいたい0.881です。

みなみ先生：正解。予測確率が0.881で、0.5以上だから？

たくみ：（自信ありげに）陽性に分類されます！

みなみ先生：（拍手するように）その通り。次はジニ不純度と情報利得。あるノードに10件のデータがあって、陽性6件、陰性4件。これを、陽性5件・陰性0件のグループエーと、陽性1件・陰性4件のグループビーに分岐したとする。まず分岐前のジニ不純度を求めてみて。

たくみ：陽性の割合は10分の6で0.6、陰性は0.4。1引く、0.6の2乗足す0.4の2乗、だから、1引く、0.36足す0.16、つまり1引く0.52で……0.48です。

みなみ先生：正解。じゃあグループエーとグループビーはどうなる？

たくみ：グループエーは陽性だけだから、1引く1の2乗足すゼロの2乗で、ジニ不純度はゼロ。グループビーは、陽性の割合が0.2、陰性が0.8だから、1引く、0.04足す0.64、つまり1引く0.68で、0.32です。

みなみ先生：じゃあ、分岐後の重み付き平均は？

たくみ：どちらも5件ずつだから、2分の1かけるゼロ足す、2分の1かける0.32で……0.16です。

みなみ先生：情報利得は、分岐前引く分岐後だから？

たくみ：0.48引く0.16で、0.32です！

みなみ先生：（満足げに）バッチリ。じゃあ最後にk近傍法。訓練データが6個あって、点エー、座標は1と1で、ラベルは陽性。点ビーは2と1で陽性。点シーは1と2で陽性。点ディーは5と5で陰性。点イーは6と5で陰性。点エフは5と6で陰性。ここに、新しい点キュー、座標2と2、をケー3のk近傍法で分類したい。

たくみ：まず、キューから各点までの距離を出すんですよね。

みなみ先生：そう。エーまでの距離は、横に1、縦に1離れてるから、1の2乗足す1の2乗のルート、つまりルート2で、だいたい1.414。ビーまでは、横に0、縦に1だから、距離はちょうど1。シーまでも同じく距離1。ディー、イー、エフはずっと遠くて、それぞれだいたい4.243、5、5になる。

たくみ：じゃあ、近い順に3つ選ぶと、ビーとシーが1で並んで、その次がエーの1.414。

みなみ先生：その3点のラベルは？

たくみ：ビーもシーもエーも、全部「陽性」です！

みなみ先生：じゃあ多数決の結果は？

たくみ：（納得した声で）全員陽性だから、キューも陽性に分類されます。

みなみ先生：（拍手するように）完璧。3つとも実際に手を動かして解けたね。

たくみ：（達成感のある声で）数字を追いかけると、それぞれの手法の考え方の違いがよく分かりました。

みなみ先生：（少し真剣な声で）じゃあ、つまずきやすいポイントも話しておこうか。たくみくん、分類の問題で失敗したことある？

たくみ：（気まずそうに）あります……。ロジスティック回帰の閾値0.5を、絶対のルールだと思い込んでました。

みなみ先生：あるあるだね。閾値0.5はあくまで一例で、例えば病気の見逃しを避けたい場面では、閾値を下げて見逃しを減らす、みたいな調整をすることもあるんだ。

たくみ：（反省した様子で）状況によって変えていいものなんですね。あと、決定木も、分岐を増やせば増やすほど賢くなると思ってました。

みなみ先生：それも典型的な誤解。決定木は深く育てすぎると、訓練データを丸暗記するみたいに過学習してしまう。だから木の深さの上限とか、葉に必要な最小データ数とかで複雑さを制限するの。

たくみ：（納得した声で）育てすぎもよくないんですね。

みなみ先生：それと、k近傍法も同じで、ケーが小さいほど過学習寄り、大きいほど未学習寄り。しかも距離を使う手法だから、特徴量の標準化を忘れると、値の範囲が大きい特徴量だけで距離がほとんど決まっちゃう、っていう落とし穴もある。

たくみ：（ヒヤリとした声で）それ、うっかりやりそうです……。

みなみ先生：最後に一つ。「バギングはバリアンスを下げる」「ブースティングはバイアスを下げる」っていうのは、あくまで主な傾向であって、単純化しすぎないこと。どちらも仕組みは違うけど、目指してるのは汎化性能の向上なんだ。

たくみ：（しっかりと）分かりました。傾向として覚えつつ、決めつけすぎないようにします。

みなみ先生：（優しく）じゃあ、今日のまとめ。分類では、ロジスティック回帰がシグモイド関数で確率を出力して、交差エントロピー損失で学習する。決定木はジニ不純度と情報利得を使って賢く枝分かれする。k近傍法は距離が近いケー個の多数決というシンプルな仕組みで、ケーの大きさがバイアスとバリアンスのトレードオフを生む。そして、これらを組み合わせるアンサンブル学習という考え方もある。

たくみ：（満足そうに）今日も盛りだくさんでした。ありがとうございます。

みなみ先生：（にっこり）次回は、教師あり学習から離れて「教師なし学習」、クラスタリングと次元削減の話に入るよ。正解ラベルがないデータから、データ自身に隠れた構造を見つけ出す、また違った面白さがあるからお楽しみに。

たくみ：（目を輝かせて）正解がないのに学習するって、想像つかないです。楽しみにしてます！

みなみ先生：（笑顔で）それじゃあ、また次回お会いしましょう。
