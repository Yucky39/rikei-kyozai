# 番組「みなみ先生の数学・機械学習ラジオ」制作ガイド

高校数学I〜III・A〜C、そして機械学習までを、先生と生徒の対話形式で学ぶポッドキャスト用トークスクリプト集です。音声合成（NotebookLMなどのTTS）にそのまま読み込ませることを想定しているため、**数式記号は一切使わず、すべて話し言葉に変換**します。

## 登場人物

- **みなみ先生**：明るく面倒見のいい数学の先生。生徒の疑問に付き合いながら、具体例と直感的なたとえ話で説明する。時々冗談を言ったり、生徒の発見を大げさに褒めたりする。
- **たくみ**：好奇心はあるが数学に少し苦手意識のある高校生。素朴な疑問をぶつけたり、勘違いをしたり、分かった瞬間に興奮したりする。テスト前は少し焦っている。

この2人の関係性・口調は全話を通して統一してください（初対面のような自己紹介は最初の回以外では不要）。

## 台本フォーマット

- 各行は `みなみ先生：` または `たくみ：` から始める発話形式。
- 感情・トーンのト書きは話者名の直後に丸括弧で入れる。例：`みなみ先生：（笑いながら）` `たくみ：（驚いた声で）` `たくみ：（少し不安そうに）`
- ため息・沈黙・食い気味のツッコミなど、間や感情の起伏を表す短いト書きを自然な頻度で入れる（多用しすぎない。1〜2行に1回程度が目安）。
- 数式・記号は**厳禁**。$x^2$ は「エックスの2乗」、$\frac{1}{2}$ は「2分の1」、$\sqrt{2}$ は「ルート2」、$\sin\theta$ は「サインシータ」のように、すべて耳で聞いて分かる自然な日本語の話し言葉に変換すること。図やグラフの説明も、言葉だけで情景が浮かぶように描写する。

## 各回の構成（目安）

1. **オープニング**：番組名に軽く触れつつ、今日のテーマを自然に切り出す（毎回同じ導入文の丸暗記にならないよう変化をつける）
2. **今日学ぶこと**：この単元で何ができるようになるかを、たくみの疑問や日常の例と絡めて提示
3. **本編（対話で概念を解説）**：一方的な説明ではなく、たくみが質問し、勘違いし、みなみ先生が例え話や具体例で修正し、たくみが「あ、分かった！」と納得する、というやり取りを繰り返す。単元の核心的な考え方・公式の意味を、必ず「なぜそうなるのか」を含めて話し言葉で説明する。
4. **具体例を声で解く**：数値を使った具体例を1つ以上、声に出して追える形で一緒に解く（暗算できる規模の数字を使う）
5. **つまずきやすいポイント**：多くの生徒がやりがちな間違いを、たくみ自身の失敗談・ヒヤリとした体験として自然に盛り込む
6. **クロージング**：今日のまとめと、次回予告（次の単元名を軽く匂わせる）

## 分量の目安

- 1話あたり、ト書きを除く発話本文でだいたい2500〜4000文字程度（日本語の話し言葉として6〜10分程度の尺を想定）。
- 短すぎて内容が薄くならないよう、必ず本編で「なぜ」を掘り下げ、具体例を声で解く時間を確保すること。

## 正確性

内容は必ず指定された既存教材（`math/`・`machine-learning/` 配下の該当ファイル・該当行範囲）の解説・例題・つまずきやすいポイントに基づくこと。教材にない話を創作して数学的に誤った説明をしないこと。数値例も教材にあるものを話し言葉に変換するか、教材の考え方に忠実な新しい具体例を使う（勝手に不正確な近似値や誤った公式を作らない）。

## ファイル構成

`podcast/<course>/<連番>-<unit-slug>.md` の形式。各ファイルの冒頭に以下の1行メタ情報を書く（Jekyllのfront matterではない、単なる見出し）。

```
# 第◯回：〈単元名〉（〈科目名〉）
```

その直後から台本本文を書く。
